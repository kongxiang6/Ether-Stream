# Ether Stream

Ether Stream 是一套基于原始以太网帧传输 JPEG 画面的低延迟发射端 / 接收端工具。

它的工作方式：

- 发射端：抓屏 -> JPEG 编码 -> 分片 -> 原始以太网发送
- 接收端：抓取原始以太网帧 -> 入队 -> 重组 -> UDP 转发

说明：

- 接收端的 UDP 转发内容是“每个 UDP 包一张完整 JPEG”
- 这不是 RTSP / RTP / 标准视频流
- 如果第三方预览程序不是按这个格式解码，就会看不到画面
- 如果第三方程序绑定的是本机物理网卡 IP，而不是 `127.0.0.1`，那 UDP 转发地址也要填写对应网卡 IP

## 一、当前版本功能

- 自定义目标 MAC 定向发送
- 发射端双缓冲，抓屏和发送互不阻塞
- 接收端抓包回调只负责入队，重组和转发在独立线程完成
- 接收端内置实时预览窗口，默认关闭，可按需开启
- 队列满时自动丢最旧数据，避免卡死
- 支持片段超时清理
- 发射端 GUI 已中文化
- 网卡列表支持自动排序和推荐显示
- 发射端采集区域支持“屏幕中心”与“自由移动”两种模式
- 发射端采集区域最大限制为 `640x640`
- 发射端采集帧率最大限制为 `360`
- 发射端支持“锁定 JPEG 质量”，默认推荐开启
- 发射端实时详情可显示设定帧率、实采/实发覆盖率、采集/编码/发包耗时、旧帧丢弃
- 发射端界面已去掉“源 MAC”输入项
- GUI 参数会自动保存，下次打开会自动恢复

## 二、运行原理

协议固定使用：

- EtherType：`0x88B5`
- 分片头结构：`!IHHHQ`

载荷结构如下：

```text
frame_id      uint32
frag_index    uint16
frag_total    uint16
payload_len   uint16
sent_ts_ms    uint64
jpeg_chunk    bytes
```

默认单帧负载预算为 `1400` 字节，包含分片头。

## 四、命令行用法

列出可用网卡：

```powershell
python .\sender.py --list-ifaces
python .\receiver.py --list-ifaces
```

启动接收端：

```powershell
python .\receiver.py --iface "\\Device\\NPF_{YOUR-IFACE-GUID}" --udp-target 127.0.0.1:4455
```

启动发射端：

```powershell
python .\sender.py --iface "\\Device\\NPF_{YOUR-IFACE-GUID}" --dst-mac 00:11:22:33:44:55 --fps 60 --quality 80
```

- JPEG 质量、帧率、负载预算调节

- 中文悬浮说明
- 中文日志和状态显示

## 六、发射端参数说明

`网卡接口`

- 选择当前发包使用的网卡
- 程序会把更可能正确的实体网卡排在前面并标记 `【推荐】`

`目标 MAC`

- 必填
- 填接收端网卡的 MAC 地址
- 格式例如：`80:fa:5b:60:12:34`

`采集帧率`

- 每秒抓取多少帧
- 默认：`60`
- 最大：`360`
- 常用：`60`
- 可选更高帧率，例如：`120`、`150`、`300`、`360`
- 帧率越高，对 CPU、编码、网卡和接收端解码压力越大，不一定更流畅
- 如果你设置了 `240` 但实际只跑到 `180` 左右，优先看发射端实时详情里的“实采”“实发”“旧帧丢弃”“采集/编码/发包耗时”
- 一般来说：`实采`低，说明卡在抓屏；`实采`高但`实发`低，说明卡在编码或发包；`旧帧丢弃`持续增加，说明程序正在主动丢旧帧以换低延迟

`JPEG 质量`

- 画面压缩质量
- 越大越清晰，但占用越高
- 常用：`70` 到 `85`
- 自适应模式只会在你填写的质量上限以内调整，不会再超过你手填的数值

`质量模式`

- 默认推荐勾选 `锁定 JPEG 质量`
- 开启后，会尽量固定使用你填写的质量值
- 只有在单帧 JPEG 过大、可能超过安全发送大小时，才会临时压低当前帧质量
- 关闭后，程序会在你填写的质量上限以内做自适应，优先保证发送稳定性和低延迟

`单帧负载预算`

- 每个以太网分片装多少数据
- 默认：`1400`
- 一般保持默认即可

`采集区域`

- 填一个数字即可
- 例如填 `320`，表示抓屏幕正中间 `320x320`
- 最大：`640`
- 不能留空抓全屏，当前版本固定为正方形区域采集

`截取位置`

- 默认是 `屏幕中心`
- 也可以切换到 `自由移动`
- 切换后会出现一个和采集区域同尺寸的方框
- 把方框拖到目标位置后点击确认，后续发送的就是这个区域

## 七、接收端参数说明

`监听网卡`

- 选择接收数据的网卡
- 一般选和发射端连接的那张网卡

`UDP 转发地址`

- 格式：`IP:端口`
- 例如：`127.0.0.1:4455`
- 选择监听网卡后，会自动带出该网卡的本机 IPv4 地址，并保留端口号

`队列长度`

- 默认：`4096`

`拼包超时(秒)`

- 默认：`0.25`
- 同一帧分片在超时时间内没收齐就直接丢弃
- 数值太小容易丢帧，太大容易让旧帧排队

- 队列长度：`4096`

发射端：

- 采集帧率：`60`

- 质量模式：勾选 `锁定 JPEG 质量`
- 单帧负载预算：`1400`
- 采集区域：`320`

## 十一、常见问题

`invalid MAC address: ''`

- 原因：目标 MAC 没填
- 处理：在发射端填写接收端网卡 MAC

`只拷 exe 无法运行`

- 原因：缺少 `_internal`
- 处理：整个文件夹一起拷贝

`检测不到网卡`

- 原因：通常未安装 Npcap，或者没有管理员权限
- 处理：安装 Npcap，并用管理员身份运行程序

`网卡太多，不知道选哪个`

- 现在 GUI 已支持推荐网卡
- 优先选择带 `【推荐】` 的那一项
