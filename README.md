# Ether Stream

Ether Stream 是一套基于原始以太网帧传输 JPEG 画面的低延迟发射端 / 接收端工具。

它的工作方式与您提供的同类工具一致：

- 发射端：抓屏 -> JPEG 编码 -> 分片 -> 原始以太网发送
- 接收端：抓取原始以太网帧 -> 入队 -> 重组 -> UDP 转发

说明：

- 接收端的 UDP 转发内容是“每个 UDP 包一张完整 JPEG”
- 这不是 RTSP / RTP / 标准视频流
- 如果第三方预览程序不是按这个格式解码，就会看不到画面
- 如果第三方程序绑定的是本机物理网卡 IP，而不是 `127.0.0.1`，那 UDP 转发地址也要填写对应网卡 IP

## 一、当前版本功能

- 自定义目标 MAC 定向发送
- 发射端双缓冲，抓屏和发送互不阻塞
- 接收端抓包回调只负责入队，重组和转发在独立线程完成
- 接收端内置实时预览窗口，收到完整 JPEG 后可直接在界面查看
- 队列满时自动丢最旧数据，避免卡死
- 支持片段超时清理
- 发射端 GUI 已中文化
- 网卡列表支持自动排序和推荐显示
- 发射端采集区域已改为“屏幕中心正方形区域”
- 发射端界面已去掉“源 MAC”输入项
- GUI 参数会自动保存，下次打开会自动恢复

## 二、运行原理

协议固定使用：

- EtherType：`0x88B5`
- 分片头结构：`!IHHHQ`

载荷结构如下：

```text
frame_id      uint32
frag_index    uint16
frag_total    uint16
payload_len   uint16
sent_ts_ms    uint64
jpeg_chunk    bytes
```

默认单帧负载预算为 `1400` 字节，包含分片头。

## 四、命令行用法

列出可用网卡：

```powershell
python .\sender.py --list-ifaces
python .\receiver.py --list-ifaces
```

启动接收端：

```powershell
python .\receiver.py --iface "\\Device\\NPF_{YOUR-IFACE-GUID}" --udp-target 127.0.0.1:4455
```

启动发射端：

```powershell
python .\sender.py --iface "\\Device\\NPF_{YOUR-IFACE-GUID}" --dst-mac 00:11:22:33:44:55 --fps 60 --quality 80
```

## 六、发射端参数说明

`网卡接口`

- 选择当前发包使用的网卡
- 程序会把更可能正确的实体网卡排在前面并标记 `【推荐】`

`目标 MAC`

- 必填
- 填接收端网卡的 MAC 地址
- 格式例如：`80:fa:5b:60:12:34`

`采集帧率`

- 每秒抓取多少帧
- 默认：`60`
- 常用：`60`
- 可选更高帧率，例如：`120`、`150`、`300`
- 帧率越高，对 CPU、编码、网卡和接收端解码压力越大，不一定更流畅

`JPEG 质量`

- 画面压缩质量
- 越大越清晰，但占用越高
- 常用：`70` 到 `85`

`单帧负载预算`

- 每个以太网分片装多少数据
- 默认：`1400`
- 一般保持默认即可

`采集区域`

- 填一个数字即可
- 例如填 `320`，表示抓屏幕正中间 `320x320`
- 留空表示抓整个屏幕
- 如果发射端窗口本身放在屏幕正中间，采集画面里也会把发射端窗口一起采进去

## 七、接收端参数说明

`监听网卡`

- 选择接收数据的网卡
- 一般选和发射端连接的那张网卡

`UDP 转发地址`

- 格式：`IP:端口`
- 例如：`127.0.0.1:4455`
- 选择监听网卡后，会自动带出该网卡的本机 IPv4 地址，并保留端口号

`队列长度`

- 默认：`4096`

`拼包超时(秒)`

- 默认：`0.25`

## 十、目标机使用方法

在其它电脑上使用时：

1. 先安装 `Npcap`
2. 解压整个发布包
3. 保持目录结构不变
4. 运行：
   - `接收端\接收端.exe`
   - `发射端\发射端.exe`

1. 先安装 `Npcap`
2. 解压 `Ether-Stream-单文件版.zip`
3. 可直接运行：
   - `接收端-单文件版.exe`
   - `发射端-单文件版.exe`

## 十一、常见问题

`invalid MAC address: ''`

- 原因：目标 MAC 没填
- 处理：在发射端填写接收端网卡 MAC

`只拷 exe 无法运行`

- 原因：缺少 `_internal`
- 处理：整个文件夹一起拷贝

`检测不到网卡`

- 原因：通常未安装 Npcap，或者没有管理员权限
- 处理：安装 Npcap，并用管理员身份运行程序

`网卡太多，不知道选哪个`

- 现在 GUI 已支持推荐网卡
- 优先选择带 `【推荐】` 的那一项
